Writing Entries

To write entries all you need is a GitHub account. Entries are written in a basic text file with two sections. The first, the front matter is between two lines of three consecutive dashes --- represents datasets in simple key: value pairs written in in YAML. The second part is the markdown text written in Markdown Syntax. An entry can have several components rendered from the data in the front matter and the text in markdown. The elements of an entry page are as follows (hover over the image to remove labels):

It is important to note that the rest of the elements come from the front matter.

Text

The text comes directly and only from the second part of the file, the markdown text. Markdown is pretty simple, see this guide from Learn X in Y Minutes. Below is a small snippet:

# Heading one
Writing with _italics_ and with **bold** syntax. Create links by enclosing the word in [brackets](http://linksomewhere.com) followed by the link enclosed in parenthesis. Write in `code`.

## Heading two
Write a list:
  - one item
  - two item
    - subitem

Adding images

In case you need to format an image, table, or something else, you can add add {: .class .another} after the element. We use the Semantic-UI framework, e.g. for images:

![](/docs/tutorials/AvocadoLab/image-example.jpg){: .ui .fluid .image}

![](/docs/tutorials/AvocadoLab/image-example.jpg){: .ui .small .centered .image}

Title and subtitle

The title: should be the name of your entry. It should be short and easy if possible. The subtitle: is optional and is a short description of the entry.

Menu

The menu is set automatically. The only way to add additional items is to edit other entry pages and specify host-org: exactly as the title: of this entry. For example, if ‘Avocado Lab’ wants to list Projects, an entry for the Project Guacamole should be created and specify host-org: Avocado Lab.

Header and logo

To display a header image at the top of your entry, you can either save an image in the folder of your initiative with the name ‘header’ in .png or .jpg format, or add the ‘header’ key in the front matter with a link to the image, e.g. header: http://linktoheaderimage.jpg

The same goes for the logo, which is displayed inside the infobox and when clicked links to the entry’s website.

You can also add a thumbnail image to the folder. Although it is not displayed in the entry, it is meant to be showed in the List of Entries page. The page grabs images in this order: thumbnail, logo, header, a dummy image.

Infobox

The infobox is the box on the left hand side of the entry. It can show several data, the following table shows the key, description, and an example in order of how they would appear. The green rows represent mandatory keys, while the brown rows are mandatory for some collections.

Key Description Example
predecessor: Predecessor is the previous name of the initiative. Sometimes initiatives change names, Groups evolve into Labs, Events can have past conferences, etc. predecessor: Avocado Group
succesor: Successor is the next, or upcoming initiative. successor: Avocado Empire
status: Status is automatically 'active' and will not be shown. Other options are: 'closed', 'planning', 'building', 'reformatting' status: Planning
start-date: Start Date should be a year, or can also include month and day. It should follow the ISO format of 'YYYY-MM-DD' start-date: 2008-04-26
end-date: End Date only applicable to 'closed' initiatives, and is mandatory for events end-date: 2017
host-org: Host is the organization, community or space that runs or organizes the initiative. A Lab hosted by a hackerspace, a Project hosted by a Lab, or a Startup hosted by an Incubator, etc. host-org: '[Tree Labs](http://treelabs.com)'
type-org: TypeDefines the organization, purpose, or sponsor behind the initiative. There are eight to choose from: Community (grassroots), School (primary and secondary), University (higher education), Non-Profit (or NGO's), For-Profit, Museum, Government,or Other type-org: Community
affilliations: Affiliations can be partners, networks, sponsors, or anyone you wish to acknowledge or be linked to your entry. Try to add them as markdown links in between quotation marks 
affilliations:
  - '[Trees of the World](http://worldtrees.net)'
  - '[Yummy fruits](http://yummyfruits.org)'
area: Area is useful to area: 30 m2
size: Selected size: 4 members
address: Address is the street and number of the physical location, rendered under Location address: Tropical Av.
directions: Directions are additional location pointers to arrive, rendered under Location directions: 2nd floor, green door

Social Icons

Links to any kind of website, social media, or contact information can be presented here.

Tags

You are probably familiar with tags, they are used as keywords to easily categorize entries. A good rule of thumb is to have maximum 10 tags. In addition, it is good practice to search the current tags used by other entries and reuse them if applicable.

Promotion boxes

Promotions are boxed items on the left hand side of the entry. Here you can promote whatever you want, you meetups, classes, to join the project, etc. You can add as many as you want, but a good rule of thumb is no more than 3.

To add a promotion you must write in the front matter:

promotions:
  - text: #write here your promotion
    button: #text in the button
    URL: #link for the button
    image: #path to the image in the background
  - text: # second promotion starts with another dash
    button:
    URL:
    color: #specify a background color instead of an image. wrap the color in '', e.g. '#43BB7A'

The rendered box will first try to grab the promotions.image key, then the promotions.color key. If these are not specified the background color will be grey.

Get your promotion in the landing page!

Entries can have their promotions featured in the landing page carousel if they contain a background image, or if the entry has a header image
